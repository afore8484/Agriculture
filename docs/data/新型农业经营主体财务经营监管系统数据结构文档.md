# 新型农业经营主体财务经营监管系统数据结构文档

## 4. 数据需求

### 4.1 数据分类

#### 4.1.1 静态数据
静态数据是指变化频率低、主要承担约束、配置、关联和口径统一作用的数据。本模块建议包含以下静态数据：

| 数据类别 | 说明 | 主要用途 |
|---|---|---|
| 行政区划主数据 | 省、市、县、乡、村区划编码、名称、层级、上下级关系 | 支撑主体分布、项目范围、监管统计 |
| 主体类型字典 | 合作社、家庭农场、农业企业等类型编码与说明 | 支撑主体分类统计和筛选 |
| 行业类型字典 | 种植、养殖、加工、服务等行业分类 | 支撑行业结构分析 |
| 金融产品字典 | 产品类型、授信方式、期限类型、利率类型 | 支撑金融产品标准化管理 |
| 审批状态字典 | 草稿、已提交、审核中、通过、驳回、关闭 | 支撑项目、示范、信贷全流程流转 |
| 角色权限配置 | 主体端/政府端/金融端角色与菜单权限 | 支撑三端菜单展示与权限隔离 |
| 计量与仓储字典 | 计量单位、仓库类型、出入库类型 | 支撑进销存业务标准化 |
| 政策公告分类 | 通知、政策、解读、法规分类 | 支撑政策公告治理 |

#### 4.1.2 动态数据
动态数据是指随着业务办理持续新增、更新和流转的数据。本模块建议包含以下动态数据：

| 数据类别 | 说明 | 主要用途 |
|---|---|---|
| 经营主体档案数据 | 主体基础信息、经营信息、财务报送信息 | 支撑主体端填报与政府端监管 |
| 成员账户与出资数据 | 成员信息、出资记录、成员交易 | 支撑权益核算与盈余分配 |
| 盈余量化数据 | 财政补助、捐赠财产、公积金、盈余分配 | 支撑主体经营收益管理 |
| 进销存业务数据 | 商品、客户、仓库、订单、出入库、库存 | 支撑经营交易全链路 |
| 项目申报数据 | 政府项目发布、主体申报、审批流转 | 支撑项目管理与过程监管 |
| 示范管理数据 | 示范申报、监测指标、审核结论 | 支撑示范主体管理 |
| 金融业务数据 | 金融产品、贷款申请、审核、放款、还款、逾期 | 支撑银企协同与风险监控 |
| 公告政策数据 | 通知公告、政策法规、阅读记录 | 支撑信息发布与宣贯 |
| 日志审计数据 | 登录日志、操作日志、同步日志、附件日志 | 支撑审计追溯与问题定位 |

### 4.2 数据库设计概述

#### 4.2.1 数据库类型与选型
本模块建议采用“关系型数据库 + 缓存 + 对象存储”的组合方案：

1. 业务主库：`PostgreSQL 15`（或兼容 `MySQL 8`）
- 适合复杂查询、统计聚合、多表关联和事务一致性。
- 支持 JSON 扩展字段、后续分区表扩展。

2. 缓存层：`Redis`
- 缓存区划树、字典项、热点统计、菜单权限。

3. 附件存储：`MinIO / OSS / S3`
- 存放项目申报材料、示范监测附件、政策附件、单据附件。

4. 检索增强（可选）：`Elasticsearch`
- 用于政策公告全文检索和跨字段检索。

### （补充）字段与约束建议

1. 所有业务表统一审计字段
- 建议统一补充：`created_by`,`created_at`,`updated_by`,`updated_at`,`deleted_flag`。

2. 编码唯一约束
- `entity_code`,`project_code`,`product_code`,`order_no`,`apply_no`,`doc_no` 应建立唯一索引。

3. 状态字段统一字典
- `status`,`*_status`,`review_result` 统一走字典，避免自由文本导致统计口径不一致。

4. 金额字段统一精度
- 金额字段建议 `numeric(18,2)`；数量字段建议 `numeric(18,4)`。
