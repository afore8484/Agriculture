##### 4.2.2.5 关键关系说明

1. `adm_region` 是区划主维表，所有业务表均通过 `region_code` 关联。
2. `income_ledger_header` 是业务核心主表，向下关联：
   - `income_ledger_item`
   - `income_ledger_attachment`
   - `income_ledger_flow_log`
   - `income_warning_event`
3. `income_fill_window` 控制 `income_ledger_header` 和 `gdp_growth_target` 的录入权限与时间窗口。
4. `income_warning_rule` 负责预警判断规则，`income_warning_event` 保存命中结果。
5. `info_collection_stat_snapshot`、`district_summary_snapshot`、`annual_income_stat` 均可由 `income_ledger_header`、`income_ledger_item` 和特殊区域标识表汇总生成。
6. `data_import_job`、`data_export_job`、`module_operation_log`、`sync_interface_log` 用于支撑导入导出、审计追踪和接口运维，不直接承载业务主数据。

##### 4.2.2.6 索引建议

1. `income_ledger_header`：建议建立 `(region_code, stat_year, stat_quarter)` 联合索引。
2. `income_ledger_header`：建议建立 `(report_status, audit_status)` 联合索引。
3. `gdp_growth_target`：建议建立 `(region_code, stat_year)` 唯一索引。
4. `income_warning_event`：建议建立 `(event_status, severity, generated_at)` 联合索引。
5. `district_summary_snapshot`、`annual_income_stat`：建议建立 `(region_code, stat_year)` 或 `(region_code, stat_year, stat_quarter)` 索引。
6. `adm_region`：建议对 `region_code`、`parent_code` 建索引，用于区划树查询。
