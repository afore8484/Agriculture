# 新型农业经营主体财务经营监管系统完整版接口文档


## 一、公共与首页

### 1. 获取行政区划树

- 方法：`GET`
- URL：`/api/nongjing/common/regions`
- 用途：获取行政区划层级树，用于全局筛选、级联选择和数据权限过滤。
- 入参：
  - parentCode：上级区划编码，可空
  - level：区划层级，可选（省/市/县/乡镇/村）
  - keyword：区划名称关键字，可选
- 返回：
  - regionCode：区划编码
  - regionName：区划名称
  - level：区划层级
  - parentCode：上级区划编码
  - hasChildren：是否有子节点
- 数据来源：`adm_region`

### 2. 获取数据字典项

- 方法：`GET`
- URL：`/api/nongjing/common/dicts`
- 用途：按字典类型获取字典项，用于主体类型、行业类型、审批状态、仓库类型、政策分类等下拉选项。
- 入参：
  - dictType：字典类型
  - parentCode：上级字典编码，可空
  - keyword：字典名称关键字，可空
- 返回：
  - dictCode：字典编码
  - dictName：字典名称
  - dictValue：字典值
  - parentCode：上级编码
  - sortNo：排序号
- 数据来源：`sys_dict_item`

### 3. 获取组织机构列表

- 方法：`GET`
- URL：`/api/nongjing/common/orgs`
- 用途：获取政府、银行、经营主体等组织机构信息，用于筛选、归属和审批流配置。
- 入参：
  - orgType：机构类型，可选（政府/银行/主体）
  - regionCode：所属区划，可空
  - keyword：机构名称关键字，可空
- 返回：
  - orgId：机构ID
  - orgCode：机构编码
  - orgName：机构名称
  - orgType：机构类型
  - regionCode：所属区划
- 数据来源：`sys_org`

### 4. 获取当前用户信息

- 方法：`GET`
- URL：`/api/nongjing/common/current-user`
- 用途：获取当前登录用户、所属机构、端类型和权限信息，用于页面初始化与数据范围控制。
- 入参：
  - token：登录态令牌（Header 传递）
- 返回：
  - userId：用户ID
  - username：登录名
  - realName：姓名
  - orgId：所属机构ID
  - terminalType：端类型
  - roles：角色列表
- 数据来源：`sys_user`、`sys_user_role`、`sys_role`

### 5. 获取角色列表

- 方法：`GET`
- URL：`/api/nongjing/common/roles`
- 用途：获取角色与数据权限范围，用于权限配置、用户授权及页面控制。
- 入参：
  - terminalType：端类型，可空
  - status：状态，可空
- 返回：
  - roleId：角色ID
  - roleCode：角色编码
  - roleName：角色名称
  - terminalType：端类型
  - dataScope：数据权限范围
- 数据来源：`sys_role`

### 6. 文件上传

- 方法：`POST`
- URL：`/api/nongjing/common/attachments`
- 用途：上传业务附件，用于月报、项目申报、示范监测、通知公告、贷款申请等场景。
- 入参：
  - bizType：业务类型
  - bizId：业务ID，可空
  - file：上传文件
- 返回：
  - attachmentId：附件ID
  - fileName：文件名
  - fileUrl：文件地址
  - fileSize：文件大小
  - fileType：文件类型
- 数据来源：`sys_attachment`

### 7. 获取附件列表

- 方法：`GET`
- URL：`/api/nongjing/common/attachments`
- 用途：根据业务类型和业务ID查询附件清单。
- 入参：
  - bizType：业务类型
  - bizId：业务ID
- 返回：
  - attachmentId：附件ID
  - fileName：文件名
  - fileUrl：文件地址
  - fileType：文件类型
  - uploadedAt：上传时间
- 数据来源：`sys_attachment`

### 8. 删除附件

- 方法：`DELETE`
- URL：`/api/nongjing/common/attachments/{attachmentId}`
- 用途：删除未绑定或允许删除的业务附件。
- 入参：
  - attachmentId：附件ID
- 返回：
  - attachmentId：附件ID
  - deleted：是否删除成功
- 数据来源：`sys_attachment`

### 9. 获取主体端首页统计

- 方法：`GET`
- URL：`/api/nongjing/dashboard/subject`
- 用途：获取主体端首页卡片、趋势图和经营分析数据。
- 入参：
  - entityId：主体ID
  - yearMonth：统计月份，可空
- 返回：
  - memberCount：成员数量
  - goodsCount：商品数量
  - capitalTotal：出资总额
  - specialFundTotal：专项资金总额
  - reserveFundTotal：公积金总额
  - tradeAmountTotal：交易总额
  - projectStats：项目统计
  - chartData：图表数据
- 数据来源：`stat_dashboard_snapshot`、`entity_profile`、`member_profile`、`biz_product`、`entity_special_fund`、`member_trade_txn`

### 10. 获取政府端大屏统计

- 方法：`GET`
- URL：`/api/nongjing/dashboard/gov`
- 用途：获取政府端大屏主体数量、类型占比、行业分布、财政补助等统计数据。
- 入参：
  - regionCode：区划编码
  - yearMonth：统计月份，可空
  - level：展示层级，可空（省/市/县）
- 返回：
  - subjectCount：主体总数
  - typeDistribution：主体类型占比
  - industryDistribution：行业分布
  - fundDistribution：资金分布
  - projectStats：项目统计
  - trendData：近五年趋势
- 数据来源：`stat_dashboard_snapshot`、`entity_profile`、`surplus_quant_item`

### 11. 获取金融端首页统计

- 方法：`GET`
- URL：`/api/nongjing/dashboard/bank`
- 用途：获取金融端首页金融产品、申贷、审核、放款、还款等统计。
- 入参：
  - orgCode：机构编码
  - yearMonth：统计月份，可空
- 返回：
  - productCount：产品数量
  - loanApplyCount：申贷数量
  - loanAmount：申贷金额
  - reviewCount：审核数量
  - disburseCount：放款数量
  - repayCount：还款数量
  - chartData：图表数据
- 数据来源：`stat_dashboard_snapshot`、`fin_product`、`fin_loan_apply`、`fin_loan_review`、`fin_loan_disburse`、`fin_loan_repay`
