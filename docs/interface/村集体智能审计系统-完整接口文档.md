# 村集体智能审计系统完整接口文档

> 说明：本接口文档基于现有接口文档、需求梳理和数据结构文档进行补齐整理，统一采用 REST 风格；对于原文档已存在接口，沿用既有命名与 URL 风格；对于缺失接口，按需求动作与数据表设计补齐。

## 一、公共与流程支撑（8）

### 1. 获取行政区划树

- 方法：`GET`
- URL：`/api/audit/common/regions`
- 用途：获取行政区划层级树，用于全局筛选、级联选择及审计对象范围定位。
- 入参：
  - parentCode：上级区划编码，可空
  - level：区划层级，可选（省/市/县/乡镇/村）
  - keyword：区划名称关键字，可选
- 返回：
  - regionCode：区划编码
  - regionName：区划名称
  - level：区划层级
  - parentCode：上级区划编码
  - hasChildren：是否有子节点
- 数据来源：`sys_region`

### 2. 获取组织机构树

- 方法：`GET`
- URL：`/api/audit/common/orgs`
- 用途：获取组织机构树，用于制定部门、审批部门、接收单位等选择。
- 入参：
  - parentOrgCode：上级组织编码，可空
  - orgType：组织类型，可选
  - regionCode：行政区划编码，可选
- 返回：
  - orgCode：组织编码
  - orgName：组织名称
  - orgType：组织类型
  - parentOrgCode：上级组织编码
  - status：状态
- 数据来源：`sys_org`

### 3. 获取用户列表

- 方法：`GET`
- URL：`/api/audit/common/users`
- 用途：按组织、岗位、关键字查询系统用户，用于组员选择、审批人选择、接收人选择。
- 入参：
  - orgCode：组织编码，可选
  - keyword：姓名/账号关键字，可选
  - status：状态，可选
  - pageNum：页码
  - pageSize：每页条数
- 返回：
  - userId：用户ID
  - username：登录账号
  - displayName：姓名
  - orgCode：所属组织
  - postName：岗位
  - status：状态
- 数据来源：`sys_user`

### 4. 获取数据字典项

- 方法：`GET`
- URL：`/api/audit/common/dicts/{dictType}`
- 用途：获取指定字典类型的配置项，用于状态、风险等级、文件类型、审批结果等前端下拉。 
- 入参：
  - dictType：字典类型
  - keyword：字典名称关键字，可选
- 返回：
  - dictCode：字典编码
  - dictName：字典名称
  - dictValue：字典值
  - sortNo：排序号
  - status：状态
- 数据来源：`sys_dict_item`

### 5. 上传通用附件

- 方法：`POST`
- URL：`/api/audit/common/attachments`
- 用途：上传业务附件并绑定到指定业务单据。支持计划、方案、承诺书、报告、政策法规、档案等统一附件能力。
- 入参：
  - bizType：业务类型
  - bizId：业务ID
  - file：附件文件
  - remark：附件说明，可选
- 返回：
  - attachmentId：附件ID
  - fileName：文件名
  - previewUrl：预览地址
  - downloadUrl：下载地址
  - uploadedAt：上传时间
- 数据来源：`sys_attachment`

### 6. 获取业务附件列表

- 方法：`GET`
- URL：`/api/audit/common/attachments`
- 用途：按业务类型和业务ID查询附件列表，用于详情页附件区域展示。 
- 入参：
  - bizType：业务类型
  - bizId：业务ID
- 返回：
  - attachmentId：附件ID
  - fileName：文件名
  - fileSize：文件大小
  - mimeType：文件类型
  - previewUrl：预览地址
  - downloadUrl：下载地址
  - uploadedBy：上传人
- 数据来源：`sys_attachment`

### 7. 删除业务附件

- 方法：`DELETE`
- URL：`/api/audit/common/attachments/{attachmentId}`
- 用途：删除指定业务附件，同时维护附件逻辑删除状态和删除日志。 
- 入参：
  - attachmentId：附件ID
- 返回：
  - attachmentId：附件ID
  - status：处理状态
- 数据来源：`sys_attachment`

### 8. 获取流程实例详情

- 方法：`GET`
- URL：`/api/audit/workflows/{instanceId}`
- 用途：查询流程实例、当前节点和任务处理轨迹，用于审批过程穿透查看。 
- 入参：
  - instanceId：流程实例ID
- 返回：
  - instanceId：流程实例ID
  - flowCode：流程编码
  - currentNode：当前节点
  - status：实例状态
  - taskList：任务节点列表
- 数据来源：`wf_instance`、`wf_task`
