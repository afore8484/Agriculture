# 农经一张图前端接入与运行说明

## 1. 前端代码目录

正式前端代码目录固定为：
`E:\workspaces\Agriculture\prototype\modules\nongjing-map\frontend`

## 2. 本轮落地内容

本轮已经完成以下内容：
- 新建独立前端工程，不再在外部 `src/onemap` 目录直接改造
- 建立总览页 + 7 个专题页路由
- 建立统一查询上下文
- 建立 `/api/nongjing` 服务层
- 建立 service 层合同样例兜底能力
- 建立 GIS 联动面板
- 建立开发态接口面板开关
- 建立最小测试能力

## 3. 运行命令

进入目录：
`E:\workspaces\Agriculture\prototype\modules\nongjing-map\frontend`

执行：
- `npm install`
- `npm run dev`
- `npm run build`
- `npm run test`

## 4. 环境变量

参考文件：
`E:\workspaces\Agriculture\prototype\modules\nongjing-map\frontend\.env.example`

可配置项：
- `VITE_API_BASE_URL`：后端服务地址
- `VITE_ENABLE_API_PANEL`：开发态是否显示接口面板
- `VITE_AUTH_TOKEN`：Bearer Token

## 5. 当前默认策略

- 如果未配置 `VITE_API_BASE_URL`，页面走 service 层合同样例数据
- 页面组件不再直接造数
- 所有接口路径统一按 `/api/nongjing` 组织
- GIS 作为横切能力存在于总览和专题联动中

## 6. 性能约束对应策略

已按你给的响应标准在当前前端里做了基础优化：
- 路由懒加载，减少首屏加载体积
- `react` / `recharts` 拆包，降低首包压力
- 页面数据按查询条件缓存，避免同条件重复加载
- 请求默认 3 秒超时，防止页面无上限等待
- 未配置后端地址时走 service 层合同样例，避免开发阶段长时间空等待

说明：
- 当前前端能控制的是“首屏加载、重复查询、等待上限、页面不卡顿”
- 你要求的 `1 秒 / 3 秒 / 5 秒 / 10 秒` 最终是否达标，还需要真实后端接口和真实数据量一起联调压测
