# 农经一张图后端开发执行计划

## 1. 文档定位

本文档用于把“农经一张图”后端开发拆成可执行阶段，并给出每个阶段的交付物、入口条件、出口条件和检查点，确保开发过程不跑偏。

## 2. 执行总原则

- 只处理“农经一张图”
- 以三份基准文档为主，不以原型页面随意倒推接口
- 先公共与总览，再财务，再其余专题
- 每阶段必须形成可验证产物，不允许长时间只做半成品

## 3. 分阶段执行

### 阶段一：后端工程初始化

目标：
- 建立 `modules/nongjing-map/backend` 正式工程

必须交付：
- Maven 工程
- Spring Boot 启动配置
- PostgreSQL 数据源配置
- Flyway 配置
- MyBatis-Plus 配置
- 统一返回体与异常处理
- Swagger/OpenAPI
- `.env.example`

入口条件：
- 三份基准文档已固定
- 技术栈已确认

出口条件：
- 应用能启动
- 数据库能连通
- Flyway 能执行
- Swagger 能打开

阶段检查点：
- 是否仍是单体结构
- 是否仍按 RuoYi-Vue-Pro 体系组织代码
- 是否没有引入与本模块无关的能力

### 阶段二：数据库迁移与读模型落地

目标：
- 把数据结构文档转成正式迁移脚本和查询基础表

必须交付：
- 维度表迁移脚本
- 事实表迁移脚本
- 汇总视图 / 物化视图脚本
- 导出任务表
- 初始化字典与指标定义

入口条件：
- 工程已可启动
- 数据库配置可用

出口条件：
- 文档中的主要数据表均已落库
- 迁移脚本可重复执行
- 表字段和文档一致

阶段检查点：
- 是否避免了重复建表
- 是否保留统一维度表
- 是否避免了高风险 PostgreSQL 专有空间依赖

### 阶段三：公共能力与总览接口

目标：
- 完成区划、时间、指标、字典、首页、地图统计能力

必须交付：
- 行政区划树接口
- 区划详情接口
- 时间维度接口
- 指标字典接口
- 通用字典接口
- 首页天气接口
- 首页卡片接口
- 首页地图统计接口

入口条件：
- 数据库基础表可查

出口条件：
- 前端总览页可改接真实公共接口
- 总览页最小联调链路可跑通

阶段检查点：
- 返回字段是否和接口文档一致
- 地图统计是否仍属于横切能力，不是独立主题

### 阶段四：财务一张图接口

目标：
- 优先完成接口数量最多、业务最重的财务专题

必须交付：
- 财务总览摘要
- 财务趋势
- 收入分档统计
- 收入分档趋势
- 高于 / 低于阈值明细
- 分档地图分布
- 财务指标明细
- 资产负债分析
- 收益分配结构
- 导出任务支持

入口条件：
- 公共能力已完成
- 财务相关事实表已可用

出口条件：
- 财务页主链路能联调
- “未建账套、未结账、收入分档、资产负债”四条主线完整

阶段检查点：
- 是否覆盖 5万/10万/20万 三档
- 是否支持明细与穿透
- 是否支持趋势与导出

### 阶段五：其余 6 个专题接口

顺序固定：
1. 资产
2. 资源
3. 合同
4. 成员
5. 股权
6. 预警监督

每个专题必须交付：
- 汇总接口
- 分类 / 结构接口
- 区域对比或排名接口
- 明细列表接口
- 必要导出接口

专题特别要求：
- 资产：区域价值排名前5
- 资源：资源信息调取相关接口
- 合同：本期 + 累计应收 / 应付
- 成员：成员结构 + 台账查询
- 股权：成员股权映射
- 预警：规则、通知、送办、受理、办结、效能闭环

出口条件：
- 68 个标准接口全部具备后端实现
- 前端 7 个专题页都有真实接口可接

阶段检查点：
- 是否存在“页面有需求，后端无接口”的遗漏
- 是否存在“接口有名字，字段却对不上”的口径漂移

### 阶段六：导出、缓存与性能治理

目标：
- 保证接口性能不违背当前性能标准

必须交付：
- 异步导出任务机制
- 高频接口缓存策略
- 慢查询识别与索引优化
- 接口耗时统计

性能目标：
- 简单查询平均响应时间：1 秒
- 复杂查询平均响应时间：3 秒
- 单条记录交易接口平均响应时间：1 秒
- 多条记录（100 条）交易接口平均响应时间：3 秒
- 批量前台经办业务数据导入（2000 条）平均 5 秒，峰值 10 秒

出口条件：
- 接口压测结果可出报告
- 高频接口具备稳定性能表现

阶段检查点：
- 是否仍采用无阻塞前端体验设计
- 是否避免长耗时同步接口直接拖垮页面加载

### 阶段七：联调与验收闭环

目标：
- 用当前前端工程完成真实联调，并校验三份基准文档一致性

必须交付：
- 联调问题清单
- 字段偏差修正清单
- 接口完成清单
- 最终运行说明

入口条件：
- 前端工程已可运行
- 后端 68 接口已基本具备

出口条件：
- 前端不再依赖合同样例数据
- 页面、接口、数据结构三层口径闭合

阶段检查点：
- 是否仍然只围绕“农经一张图”
- 是否还有 mock 未替换
- 是否还有导出、趋势、明细类接口缺失

## 4. 执行不偏离机制

每阶段执行前必须确认：
- 当前阶段的目标是否明确
- 当前阶段的交付物是否可验证
- 当前阶段是否会引入超出模块范围的新需求

每阶段执行后必须确认：
- 本阶段出口条件是否达成
- 是否出现与计划罗盘冲突的内容
- 是否需要回到文档层修正口径

## 5. 阶段产出物目录建议

建议后端模块最终至少包含：
- `src/main/java`
- `src/main/resources/db/migration`
- `src/test/java`
- `docs/`
- `.env.example`
- `README.md`

## 6. 执行纪律

- 不允许跳阶段硬做后面专题
- 不允许先写前端需要什么就临时补什么
- 不允许为了赶进度牺牲 PostgreSQL/金仓兼容性
- 不允许在未完成阶段出口条件时强行进入下一阶段

## 7. 最终用法

后续无论是人执行还是 Agent 执行，都必须：
- 先看《计划罗盘》确认方向
- 再看《执行计划》确认阶段
- 再回到三份基准文档确认内容

只有这三层同时一致，执行才算没有偏离。
