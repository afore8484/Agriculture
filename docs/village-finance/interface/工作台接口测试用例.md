# 工作台（Dashboard）接口清单与测试用例

更新时间：2026-03-21  
页面归属：前端工作台首页（`Dashboard`）

## 1. 接口清单

说明：所有接口返回统一结构 `CommonResult`，成功时 `code=0`、`message=success`。

### 1.1 工作台主接口

1. `GET /api/village-finance/home/stats`（兼容：`GET /api/home/stats`）
   - 用途：统计卡片（收入、支出、资产、待办、紧急待办）
   - 入参：`ledgerId?` `orgId?` `periodId?` `period?(yyyy-MM)` `userId?`
   - 出参：`ledgerId` `periodId` `incomeAmount` `expenseAmount` `assetAmount` `pendingTodoCount` `urgentTodoCount`

2. `GET /api/village-finance/home/todos`（兼容：`GET /api/home/todos`）
   - 用途：首页待办列表
   - 入参：`userId`(必填) `limit?`
   - 出参：`todoId` `todoType` `title` `createTime` `taskId` `instanceId` `bizType` `bizId` `nodeName` `taskStatus` `urgency`

3. `GET /api/village-finance/home/charts`（兼容：`GET /api/home/charts`）
   - 用途：月度收支趋势图
   - 入参：`ledgerId?` `orgId?` `periodId?` `period?(yyyy-MM)` `months?`
   - 出参：数组，元素字段：`periodLabel` `incomeAmount` `expenseAmount`

4. `GET /api/village-finance/home/asset-distribution`
   - 用途：资产分类分布
   - 入参：`ledgerId?` `orgId?`
   - 出参：数组，元素字段：`categoryName` `amount`

5. `GET /api/village-finance/home/recent-vouchers`
   - 用途：最近凭证列表（首页表格）
   - 入参：`ledgerId?` `orgId?` `periodId?` `period?(yyyy-MM)` `limit?`
   - 出参：`voucherId` `voucherNo` `voucherDate` `summary` `amount` `direction` `auditStatus` `voucherType` `bizId`

6. `GET /api/village-finance/home/progress`（兼容：`GET /api/home/progress`）
   - 用途：做账进度
   - 入参：`ledgerId?` `orgId?` `periodId?` `period?(yyyy-MM)`
   - 出参：`voucherTotal` `auditedTotal` `percent`

### 1.2 首页依赖接口（已补齐字段）

1. `GET /api/village-finance/vouchers`
   - 本轮补齐字段：`summary`
   - 关键入参：`ledgerId`(必填) `periodId?` `voucherNo?` `auditStatus?` `startDate?` `endDate?` `pageNum?` `pageSize?`

## 2. 测试前置条件

1. 已存在可用账套（`fin_book`）与期间（`fin_period`）。
2. 已存在凭证、审批任务、资产卡片等基础数据（可用 mock 数据）。
3. 服务已启动，默认后端地址示例：`http://localhost:48080`

## 3. 测试用例

## 3.1 `GET /home/stats`

1. `DASH-STAT-001` 正常查询（按账套+期间）
   - 请求：`GET /api/village-finance/home/stats?ledgerId=1&periodId=3&userId=100`
   - 预期：`code=0`，`data` 包含统计字段，`ledgerId=1`、`periodId=3`

2. `DASH-STAT-002` 仅传 `orgId` 自动解析账套
   - 请求：`GET /api/village-finance/home/stats?orgId=10&userId=100`
   - 预期：`code=0`，返回可用 `ledgerId/periodId`

3. `DASH-STAT-003` `period` 格式非法
   - 请求：`GET /api/village-finance/home/stats?ledgerId=1&period=2026/03`
   - 预期：`code!=0`，提示期间格式错误

4. `DASH-STAT-004` 账套不存在
   - 请求：`GET /api/village-finance/home/stats?ledgerId=999999`
   - 预期：`code!=0`，提示账套不存在

## 3.2 `GET /home/todos`

1. `DASH-TODO-001` 正常查询待办
   - 请求：`GET /api/village-finance/home/todos?userId=100&limit=10`
   - 预期：`code=0`，返回数组，元素包含 `todoId/title/taskStatus/urgency`

2. `DASH-TODO-002` 缺少必填 `userId`
   - 请求：`GET /api/village-finance/home/todos`
   - 预期：参数校验失败（HTTP 4xx 或 `code!=0`）

3. `DASH-TODO-003` `limit` 超过上限
   - 请求：`GET /api/village-finance/home/todos?userId=100&limit=999`
   - 预期：`code=0`，服务按上限截断返回（不报错）

## 3.3 `GET /home/charts`

1. `DASH-CHART-001` 正常查询趋势（默认 months）
   - 请求：`GET /api/village-finance/home/charts?ledgerId=1&periodId=3`
   - 预期：`code=0`，返回趋势数组，元素含 `periodLabel/incomeAmount/expenseAmount`

2. `DASH-CHART-002` 指定 `months=12`
   - 请求：`GET /api/village-finance/home/charts?ledgerId=1&periodId=3&months=12`
   - 预期：`code=0`，返回 12 条周期数据

3. `DASH-CHART-003` `months` 超上限
   - 请求：`GET /api/village-finance/home/charts?ledgerId=1&periodId=3&months=100`
   - 预期：`code=0`，服务按上限处理，不报错

4. `DASH-CHART-004` 非法 `period`
   - 请求：`GET /api/village-finance/home/charts?ledgerId=1&period=202603`
   - 预期：`code!=0`

## 3.4 `GET /home/asset-distribution`

1. `DASH-ASSET-001` 正常查询分布
   - 请求：`GET /api/village-finance/home/asset-distribution?ledgerId=1`
   - 预期：`code=0`，返回 `categoryName/amount`

2. `DASH-ASSET-002` 无资产数据
   - 请求：`GET /api/village-finance/home/asset-distribution?ledgerId=2`
   - 预期：`code=0`，`data=[]`

## 3.5 `GET /home/recent-vouchers`

1. `DASH-VOUCHER-001` 正常查询最近凭证
   - 请求：`GET /api/village-finance/home/recent-vouchers?ledgerId=1&limit=5`
   - 预期：`code=0`，返回不超过 5 条，字段包含 `summary/direction/auditStatus`

2. `DASH-VOUCHER-002` 按期间查询
   - 请求：`GET /api/village-finance/home/recent-vouchers?ledgerId=1&periodId=3`
   - 预期：`code=0`，结果限定在该期间

3. `DASH-VOUCHER-003` `period` 格式非法
   - 请求：`GET /api/village-finance/home/recent-vouchers?ledgerId=1&period=2026_03`
   - 预期：`code!=0`

## 3.6 `GET /home/progress`

1. `DASH-PROGRESS-001` 正常查询做账进度
   - 请求：`GET /api/village-finance/home/progress?ledgerId=1&periodId=3`
   - 预期：`code=0`，返回 `voucherTotal/auditedTotal/percent`

2. `DASH-PROGRESS-002` 仅传 `orgId` 自动解析
   - 请求：`GET /api/village-finance/home/progress?orgId=10`
   - 预期：`code=0`

3. `DASH-PROGRESS-003` 非法期间格式
   - 请求：`GET /api/village-finance/home/progress?ledgerId=1&period=26-03`
   - 预期：`code!=0`

## 3.7 兼容路径校验

1. `DASH-COMPAT-001` `/api/home/stats` 与 `/api/village-finance/home/stats` 一致性
   - 请求：分别调用两条路径，同参
   - 预期：返回业务数据一致（允许时间戳类字段存在微差）

2. `DASH-COMPAT-002` `/api/home/progress` 与 `/api/village-finance/home/progress` 一致性
   - 请求：分别调用两条路径，同参
   - 预期：`voucherTotal/auditedTotal/percent` 一致

## 3.8 依赖接口校验（凭证列表摘要）

1. `DASH-DEP-001` 凭证列表返回 `summary`
   - 请求：`GET /api/village-finance/vouchers?ledgerId=1&pageNum=1&pageSize=10`
   - 预期：每条记录包含 `summary` 字段

2. `DASH-DEP-002` 无分录摘要场景
   - 请求：同上（命中无摘要分录数据）
   - 预期：`summary` 允许为空，不影响整体返回

## 4. 示例请求（curl）

```bash
curl -G "http://localhost:48080/api/village-finance/home/stats" \
  --data-urlencode "ledgerId=1" \
  --data-urlencode "periodId=3" \
  --data-urlencode "userId=100"

curl -G "http://localhost:48080/api/village-finance/home/todos" \
  --data-urlencode "userId=100" \
  --data-urlencode "limit=10"

curl -G "http://localhost:48080/api/village-finance/home/recent-vouchers" \
  --data-urlencode "ledgerId=1" \
  --data-urlencode "limit=5"
```
