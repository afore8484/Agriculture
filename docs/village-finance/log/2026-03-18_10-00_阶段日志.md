# village-finance 阶段日志

- 时间：2026-03-18 10:00-10:45
- 模块：村委财务事务管理系统
- 范围：首轮闭环最终冲刺 C1（内部转账 -> 自动生成凭证）

## 本轮范围

1. 仅实现内部转账成功后自动生成凭证并回写 `voucherId` 的最小闭环。
2. 不扩展到账户维护增强、日记账重构、对账重构、月结重构、报表或审批流。

## 已补齐闭环点

1. `FinanceFundsServiceImpl#createInternalTransfer` 在转账成功后调用现有 `voucherService.createVoucher(...)`。
2. 凭证创建成功后，回写 `fin_internal_transfer.voucher_id`。
3. 转账入库、凭证生成、`voucherId` 回写位于同一 `@Transactional(rollbackFor = Exception.class)` 事务边界。
4. 凭证创建异常时抛出异常，内部转账不进入成功态（整体回滚语义）。
5. 凭证分录按最小规则生成：
   - 借方：转入账户 `subjectId`
   - 贷方：转出账户 `subjectId`
   - 金额：转账金额
   - 摘要：内部转账最小可读摘要

## 关键校验

1. 转账金额 > 0。
2. 转出账户与转入账户不能相同。
3. 转出、转入账户均存在且状态为 `ENABLE`。
4. 转出、转入账户必须有 `subjectId`。
5. `subjectId` 必须存在、属于同账套、且启用。
6. 按转账日期匹配会计期间，且期间不能为 `CLOSED`。

## 复用与结构说明

1. 已复用现有 `createVoucher` 能力，未重造凭证写入逻辑。
2. 本轮未新增 Flyway（沿用已存在结构）。
3. 已确认并继续沿用 `voucher_id` 关联字段。

## 测试与验证

   - 转账成功 -> 自动生成凭证并回写 `voucherId`
