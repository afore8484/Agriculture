# S2-R5 报表口径与主线规则固化说明

- 时间：2026-03-18
- 阶段：S2-R5
- 范围：仅文档收口，不新增业务功能

## 1. 账簿/报表最小口径（冻结）

1. 仅 `AUDITED` 凭证及分录参与入账统计。
2. `DRAFT` 凭证不计入余额。
3. 回收站/无效凭证不计入余额。
4. `CLOSED` 期间允许读取查询，不允许写入变更。
5. 当前账簿范围仅包含：
   - 总账（General Ledger）
   - 明细账（Detail Ledger）

## 2. 最小财务闭环能力（冻结）

1. 基础查询：账套、会计期间、会计科目树。
2. 凭证主流程：新增、修改、审核、反审核、删除、回收站。
3. 资金主流程：银行/现金账户、银行/现金日记账、内部转账。
4. 转账-凭证联动：内部转账自动生成凭证并回写 `voucherId`，同事务保障一致性。
5. 对账与期间：对账结果查询、月结、反结账与关键边界校验。
6. 账簿入口：总账、明细账最小查询。

## 3. 当前未覆盖能力

1. 科目余额表
2. 序时账
3. 辅助账（余额/明细）
4. 多栏账
5. 报表中心完整能力（统计分析、打印、导出、快照）

## 4. 下一阶段建议顺序

1. 先补账簿扩展基础：科目余额表、序时账。
2. 再补辅助账与多栏账最小查询。
3. 再补报表中心最小可用链路（查询 -> 导出 -> 打印）。
4. 最后进入二阶段模块（资产、合同、审批、银农直联、党建）。
