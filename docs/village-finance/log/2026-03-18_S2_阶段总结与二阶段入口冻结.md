# S2 阶段总结与二阶段入口冻结

- 阶段：S2-R8
- 范围：阶段收口与二阶段入口冻结（文档）

## 1. S2 阶段完成结论

1. 财务主线已完成最小稳定化：凭证、资金、期间、对账链路可运行且有回归测试支撑。
2. 账簿能力已形成最小读取体系：总账、明细账、科目余额表、序时账、辅助账（最小）、多栏账（最小）。
3. 报表最小链路已接通：查询（progress）→ 导出（export）→ 打印（print）。
4. S2 可判定为“可收口并切换二阶段入口”。

## 2. 当前冻结口径（S2-R5）

1. 仅 `AUDITED` 凭证参与入账统计。
2. `DRAFT` 凭证不计入余额。
3. 回收站/无效凭证不计入余额。
4. `CLOSED` 期间允许读取，不允许写入。

## 3. 已完成能力清单

### 3.1 财务主线

1. 账户维护（增改启停删受限）。
2. 银行/现金日记账新增与查询（含状态和期间边界）。
3. 内部转账与自动制证、`voucherId` 回写、事务一致性。
4. 凭证主流程（增改审反审删回收）及关键边界。
5. 对账、月结、反结账及关键状态迁移约束。
6. 凭证-资金链路一致性保护。

### 3.2 账簿能力

1. 总账。
2. 明细账。
3. 科目余额表。
4. 序时账。
5. 辅助账（最小）。
6. 多栏账（最小）。

### 3.3 报表最小链路

1. 进度查询（progress）。
2. 导出链路（export，最小URL返回）。
3. 打印链路（print，最小URL返回）。

## 4. 当前未覆盖能力边界

1. 辅助账明细（assist-detail）。
2. 科目汇总表（subject-summary）。
3. 报表中心三大表（资产负债表、收支表、现金流量表）及统计分析。
4. 报表导出/打印完整服务（真实文件生成、模板、审计留痕）。
5. 二阶段模块完整能力（资产、合同、审批、银农直联、党建）。

## 5. 二阶段推荐进入顺序与理由

1. 资产管理：与财务核算主链路耦合最高，优先推进可尽早形成资产-财务闭环。
2. 合同管理：与收付款、凭证联动直接相关，提前落地可降低后续接口改造成本。
3. 审批流：在资产/合同实体稳定后接入，能减少状态回滚和流程改造风险。
4. 银农直联：依赖资金规则和审批边界稳定，放在前述模块之后更稳妥。
5. 党建模块：与财务核心链路耦合度最低，最后推进对主线扰动最小。
