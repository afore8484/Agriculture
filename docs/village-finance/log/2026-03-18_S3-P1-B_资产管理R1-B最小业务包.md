# S3-P1-B 资产管理 R1-B 最小业务包（折旧/处置/盘点）

## 本轮范围
1. 资产折旧：新增、查询，最小校验（资产存在、期间可写、状态允许）。
2. 资产处置：新增、查询，最小校验（资产存在、未重复处置、期间可写）。
3. 资产盘点：盘点单新增、盘点明细新增、盘点结果查询，最小校验（资产存在、期间可写）。
4. 保持 S3-P1-A 删除保护规则一致。
5. 明确不做：资产结账/反结账、资产业务自动制证、复杂导出打印附件。

## 代码落地
1. 服务接口：
   - `FinanceAssetService` 新增折旧/处置/盘点方法。
2. 服务实现：
   - `FinanceAssetServiceImpl` 新增：
     - `createDepreciation` / `getDepreciationList`
     - `createDisposal` / `getDisposalList`
     - `createInventory` / `createInventoryDetail` / `getInventoryList` / `getInventoryResult`
   - 新增期间可写校验（按期间标签、按业务日期）。
   - 新增资产状态校验、重复处置校验、资产与盘点账套一致性校验。
3. 控制器：
   - `AssetController` 新增 8 个接口：
     - `POST /depreciation/create`
     - `GET /depreciation/list`
     - `POST /disposal/create`
     - `GET /disposal/list`
     - `POST /inventory/create`
     - `POST /inventory/detail/create`
     - `GET /inventory/list`
     - `GET /inventory/result`
4. VO：
   - 新增 `AssetInventoryDetailRespVO`
   - `AssetInventoryRespVO` 新增 `details` 字段。

## 数据结构说明
1. 本轮复用 S3-P1-A 已建表：
   - `fin_asset_depreciation`
   - `fin_asset_disposal`
   - `fin_asset_inventory`
   - `fin_asset_inventory_detail`
2. 现有表结构可满足本轮最小业务要求，本轮无新增 Flyway。

## 删除保护一致性
1. 资产卡片删除仍按 S3-P1-A 规则执行：
   - 存在折旧记录时拒绝删除；
   - 存在处置记录时拒绝删除；
   - 存在盘点明细时拒绝删除；
   - 存在统一来源模型下关联凭证时拒绝删除。

## 风险提示
1. 资产处置按“单资产单处置”最小规则落地，后续若业务要求多次处置需新增状态流转设计。
2. 期间主数据缺失时会直接拒绝写入，部署前需保证期间数据完整性。
