# S3-P1-C 资产管理 R1-C：资产业务财务联动 + 资产结账/反结账

## 本轮目标
1. 资产金额变动业务接入统一“业务来源 -> 凭证关联模型”。
2. 实现资产结账/反结账最小闭环。
3. 保持现有期间 `OPEN/CLOSED` 语义不变，不重构状态模型。

## 代码实现
1. 折旧制证联动：
   - 折旧记录入库后调用 `BizVoucherLinkService#createVoucher`。
   - 凭证来源：`ASSET_DEPRECIATION + depreciationId`。
   - 回写 `fin_asset_depreciation.voucher_id`。
2. 处置制证联动：
   - 处置记录入库后调用统一制证服务。
   - 凭证来源：`ASSET_DISPOSAL + disposalId`。
   - 回写 `fin_asset_disposal.voucher_id`。
3. 资产结账/反结账：
   - 新增资产侧接口：
     - `POST /api/village-finance/asset/period-close/close`
     - `POST /api/village-finance/asset/period-close/reopen`
   - 复用 `fin_period_close`（`close_type=ASSET`）和 `fin_period_reopen_log`（`ASSET-REOPEN-*`）。
4. 写入拦截：
   - 折旧、处置、盘点、盘点明细在写入前统一校验：
     - 会计期间未关闭；
     - 资产期间未关闭。

## 数据与迁移
1. 本轮未新增 Flyway。
2. 复用现有表结构即可落地：
   - `fin_asset_depreciation` / `fin_asset_disposal` 的 `voucher_id`。
   - `fin_period_close` 的 `close_type`。
   - `fin_period_reopen_log` 的日志记录能力。
