# S3-P2-B 阶段日志（合同管理 R1-B）

## 本轮范围
1. 合同收款最小业务能力（新增 + 查询）。
2. 合同付款最小业务能力（新增 + 查询）。
3. 合同收付款到资金日记账、凭证的最小联动。
4. 删除保护增强：存在收付款/凭证链路的合同禁止删除。

## 已完成事项
1. 新增 Flyway `V8__contract_payment_r1b.sql`，落地 `fin_contract_payment`。
2. 新增合同收付款模型与数据访问：
   - `FinContractPaymentDO`
   - `FinContractPaymentMapper`
3. 新增合同收付款接口与 VO：
   - `POST /contracts/receipt`
   - `POST /contracts/pay`
   - `GET /contracts/payment/list`
   - `GET /contracts/{contractId}/payments`
4. 服务层联动闭环：
   - 收付款记录入库；
   - 调用 `FinanceFundsService` 生成银行/现金日记账；
   - 调用 `BizVoucherLinkService` 生成合同来源凭证；
   - 回写 `journal_id`、`voucher_id`。
5. 同事务回滚保障：
   - 收付款、资金日记账、凭证、回写同事务；
   - 任一步骤失败整体回滚。

## 关键规则
1. `bankAccountId` 与 `cashAccountId` 必须且仅能传一个。
2. 收款/付款金额必须大于 0。
3. 合同必须存在、未删除、未终止且启用。
4. 业务日期必须匹配可写期间，`CLOSED` 期间拒绝写入。
5. 资金账户与合同必须同账套，且账户状态 `ENABLE`。
6. 资金账户与对方科目均需存在、启用且归属同账套。
7. 统一凭证来源：
   - 收款 `CONTRACT_RECEIPT`
   - 付款 `CONTRACT_PAYMENT`
   - `bizId=contractPaymentId`
