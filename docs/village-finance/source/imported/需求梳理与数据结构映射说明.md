# 村财务事务管理系统

## 需求梳理与数据结构映射说明

### 一、编写目的

本文档用于说明《村财务事务管理系统需求梳理》与《数据库结构设计》之间的对应关系，明确系统功能需求是否已经被数据库模型完整承载。

通过需求与数据结构的逐项映射，可验证：

- 数据库设计是否覆盖业务功能
- 是否存在漏表或冗余设计
- 各业务模块的数据边界是否清晰
- 是否具备支撑原型设计及系统开发的能力

------

## 二、总体映射原则

系统数据库设计遵循以下原则：

1. **业务对象驱动建模**

每一类业务页面或功能需求，均应对应一组核心数据表，例如：

- 凭证管理 → 凭证主表 + 分录表 + 附件表
- 资金管理 → 收支单 + 账户 + 日记账
- 合同管理 → 合同主表 + 生命周期过程表
- 在线审批 → 流程定义 + 实例 + 任务 + 历史

1. **主档数据与过程数据分离**

系统中：

- 主档表用于描述业务对象（如资产卡片、合同、党员、银行账户）
- 过程表用于描述业务行为（如折旧、审批、支付、结账）

1. **账务核算链完整性**

数据库需形成完整账务链路：

资金业务 → 凭证 → 科目 → 辅助核算 → 账簿 → 报表 → 期末处理
