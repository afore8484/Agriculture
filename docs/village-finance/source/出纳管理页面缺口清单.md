## 3. 缺失接口清单

以下接口为“页面功能点存在需求，但当前后端代码未落地或未提供等价能力”的缺口。

### 3.1 银行日记账

1. `PUT /api/village-finance/journals/bank/{journalId}`
   - 用途：编辑银行日记账
2. `DELETE /api/village-finance/journals/bank/{journalId}`
   - 用途：删除银行日记账
3. `POST /api/village-finance/journals/bank/{journalId}/audit`
   - 用途：审核银行日记账
4. `POST /api/village-finance/journals/bank/generate-voucher`
   - 用途：银行日记账生成凭证
5. `POST /api/village-finance/journals/bank/import`
   - 用途：导入银行日记账
6. `GET /api/village-finance/journals/bank/import-template`
   - 用途：下载银行日记账导入模板
7. `GET /api/village-finance/journals/bank/export`
   - 用途：导出银行日记账
8. `GET /api/village-finance/journals/bank/print`
   - 用途：打印银行日记账

### 3.2 现金日记账

1. `PUT /api/village-finance/journals/cash/{journalId}`
   - 用途：编辑现金日记账
2. `DELETE /api/village-finance/journals/cash/{journalId}`
   - 用途：删除现金日记账
3. `POST /api/village-finance/journals/cash/{journalId}/audit`
   - 用途：审核现金日记账
4. `POST /api/village-finance/journals/cash/import`
   - 用途：导入现金日记账
5. `GET /api/village-finance/journals/cash/import-template`
   - 用途：下载现金日记账导入模板
6. `GET /api/village-finance/journals/cash/export`
   - 用途：导出现金日记账
7. `GET /api/village-finance/journals/cash/print`
   - 用途：打印现金日记账
8. `POST /api/village-finance/journals/cash/generate-voucher`
   - 用途：现金日记账生成凭证

### 3.3 资金报表

1. `GET /api/village-finance/funds/account-summary`
   - 用途：账户汇总
   - 说明：接口文档已有，当前代码未见 controller 落地
2. `GET /api/village-finance/funds/category-summary`
   - 用途：收支类别汇总
   - 说明：接口文档已有，当前代码未见 controller 落地

### 3.4 出纳模板

1. `GET /api/village-finance/templates/income`
   - 用途：收入模板列表
2. `POST /api/village-finance/templates/income`
   - 用途：保存收入模板
3. `GET /api/village-finance/templates/expense`
   - 用途：支出模板列表
4. `POST /api/village-finance/templates/expense`
   - 用途：保存支出模板

### 3.5 内部转账

1. `POST /api/village-finance/internal-transfers/export`
   - 用途：导出内部转账
   - 说明：列表、新增、删除已有，导出未落地

### 3.6 出纳对账

1. `POST /api/village-finance/reconciliations/execute`
   - 用途：执行对账
2. `GET /api/village-finance/reconciliations/export`
   - 用途：导出对账单

### 3.7 初始化模板

1. `GET /api/village-finance/init-templates`
   - 用途：初始化模板列表
2. `POST /api/village-finance/init-templates`
   - 用途：保存初始化模板

## 6. 后续执行规则

后续“出纳管理”页面补齐工作按以下顺序执行：

1. 先补“页面已有动作但后端接口缺失”的能力
2. 再接“后端已存在但页面未挂接”的能力
3. 最后补“页面和接口都缺”的模板、初始化模板、报表深化能力

执行时默认使用：

1. [出纳管理页面检查基线.md](/C:/Users/yll/.codex/worktrees/e2dc/prototype/docs/village-finance/source/出纳管理页面检查基线.md)
