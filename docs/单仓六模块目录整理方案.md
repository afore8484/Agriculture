# 单仓六模块目录整理方案

## 1. 目标

当前仓库 `prototype` 继续作为总仓库使用，不拆成 6 个独立仓库。

目标是把现有内容整理成：

- 单仓管理
- 六个业务模块边界清晰
- 公共资产集中管理
- 后续支持按模块继续开发、交付、巡检和云端执行

## 2. 六个模块标准命名

建议统一使用英文短名作为目录名，中文名称仅用于文档标题。

| 中文模块名 | 英文目录名 | 说明 |
|---|---|---|
| 村委财务事务管理系统 | `village-finance` | 当前重点模块 |
| 农经一张图 | `nongjing-map` | 农经总览与图谱模块 |
| 村集体智能审计系统 | `audit-system` | 审计相关功能 |
| 村集体经济收入台账 | `income-ledger` | 收入台账模块 |
| 村集体绩效评价管理系统 | `performance-system` | 绩效评价模块 |
| 新型农业经营主体财务经营监管系统 | `finance-supervision` | 财务经营监管模块 |

## 5. 整理原则

1. 先分目录，再改引用
先把结构做清楚，再逐步修文档路径、脚本路径、自动化路径。

2. 保留共享层
以下内容不要强行归入单模块：

- `docs/初设.md`
- `docs/投标.md`
- `data/requirements/requirements_full.json`
- `data/interfaces/interface_coverage_summary*.json`
- `require/数据采集/**`
- `require/需求调研/**`

3. 模块目录只放本模块资产
模块目录内只允许放：

- 本模块需求
- 本模块原型
- 本模块接口
- 本模块测试
- 本模块 SQL
- 本模块脚本

4. 文件名允许保留中文
目录建议统一英文，文件名可以继续保持中文，降低迁移成本。

## 6. 推荐整理顺序

第一阶段：只做目录分层

- 创建模块目录和 shared 目录
- 不改文件内容
- 只移动文件到目标目录

第二阶段：修正引用

- 修正文档内链接
- 修正脚本中的硬编码路径
- 修正自动化与巡检工作目录

第三阶段：模块化执行

- 按模块独立生成文档
- 按模块独立生成原型
- 按模块独立做覆盖检查
- 按模块独立做云端任务

## 7. 下一步建议

最优先先做这 3 件事：

1. 先建立这 14 个一级目录：`modules`、`shared`、`docs/shared`、`docs/<6模块>`、`data/shared`、`require/shared`
2. 先把 `village-finance` 作为样板模块整理完
3. 样板跑通后，再按相同规则处理其余 5 个模块

当前不建议一次性移动全仓所有文件。
建议先拿 `village-finance` 做样板，确认目录、链接、脚本路径都正常，再批量扩展。
