# 村委财务事务管理系统（整改后）问题清单

## 结论
本次整改版整体可演示，但仍存在 **7 个需要处理的问题**：
- P0（阻断验收）：2 个
- P1（高优先级）：3 个
- P2（建议优化）：2 个

---

## P0（阻断验收）

### 1) 在线考试“未开始”状态无法进入开考动作
- **现象**：数据里状态为“未开始”，但按钮显示条件写的是“待开始”，导致“开始”按钮永远不出现。
- **定位**：`src/app/components/PartyBuilding.tsx:286-289,330-332`
- **影响**：在线考试流程无法闭环（组织考试后无法进入开始考试）。
- **建议**：统一状态枚举（推荐只保留 `未开始/进行中/已结束`），并修正按钮条件为 `row.status === "未开始"` 或统一改数据值。

### 2) 在线考试平均分显示异常（会出现“--分”）
- **现象**：未开始考试的 `avgScore` 被写成字符串 `"-"`，渲染逻辑又拼接 `分`，最终显示 `--分`。
- **定位**：`src/app/components/PartyBuilding.tsx:286,289,323`
- **影响**：关键业务指标展示错误，影响验收可信度。
- **建议**：`avgScore` 用 `null` 表示无数据，渲染改为 `row.avgScore == null ? "-" : \
`${row.avgScore}分\``。

---

## P1（高优先级）

### 3) 学习管理完成率计算逻辑错误
- **现象**：进度条和百分比直接使用 `completed` 人数当百分比（如 38 人显示 38%），未按 `completed/enrolled` 计算。
- **定位**：`src/app/components/PartyBuilding.tsx:261-269`
- **影响**：学习完成率指标失真，影响管理判断。
- **建议**：新增 `rate = enrolled ? Math.round((completed / enrolled) * 100) : 0`，进度条宽度与文案均使用 `rate`。

### 4) 银行账户“新增账户”只有提示，不会落表
- **现象**：弹窗“保存”仅关闭弹窗+toast，未写入 `accounts` 状态。
- **定位**：`src/app/components/BankDirect.tsx:317-325`
- **影响**：账户创建流程不可验证，验收时会判定为伪交互。
- **建议**：将输入值绑定 state，保存时 `setAccounts([newAccount, ...accounts])`。

### 5) 往来单位“新增”只有提示，不会新增记录
- **现象**：弹窗“保存”仅 toast，不更新 `units`。
- **定位**：`src/app/components/BankDirect.tsx:528-540`
- **影响**：往来单位新增流程不可验。
- **建议**：同上，保存时生成唯一 `id` 并写入 `units`。

## P2（建议优化）

### 6) 往来个人“删除”按钮无实际删除逻辑
- **现象**：点击删除只提示“确认删除？”，没有数据删除行为。
- **定位**：`src/app/components/BankDirect.tsx:464-466`
- **影响**：个人往来维护操作不完整。
- **建议**：增加 `persons` 可变状态 + 二次确认弹窗 + `setPersons` 删除。

### 7) 仍有“下一版本上线/开发中”占位入口
- **现象**：多个入口仍是占位提示。
- **定位**：
  - `src/app/components/FinanceCenter.tsx:661,724`
  - `src/app/components/BankDirect.tsx:87`
  - `src/app/components/OnlineApproval.tsx:415`
- **影响**：验收时会被归类为“功能入口存在但不可操作”。
- **建议**：至少补齐最小可演示闭环（打开弹窗、可保存本地状态、列表可见变更）。
