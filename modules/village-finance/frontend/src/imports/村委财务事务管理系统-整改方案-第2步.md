# 村委财务事务管理系统整改方案-第2步

## 1. 本步目标

在第 1 步补完交互骨架后，本步开始补“核心业务闭环”。目标是让系统至少可以演示关键业务从发起到结果的最小链路。

## 2. 本步范围

本步只做 P0 核心业务闭环：

- 财务中心闭环
- 资产管理闭环
- 合同管理闭环
- 在线审批闭环
- 银农直联查询闭环

## 3. 财务中心整改

## 3.1 出纳管理闭环

至少补通以下链路：

1. 新增银行日记账
2. 查看日记账详情
3. 编辑日记账
4. 审核日记账
5. 生成凭证
6. 从生成结果跳转到凭证详情

同时补齐：

- 现金日记账独立内容页
- 资金报表独立内容页
- 账户管理独立内容页
- 出纳模板独立内容页
- 内部转账独立内容页
- 出纳对账独立内容页

## 3.2 凭证管理闭环

至少补通以下链路：

1. 新增凭证
2. 保存后进入凭证列表
3. 查看凭证详情
4. 审核凭证
5. 反审核
6. 删除凭证
7. 在回收站看到被删除凭证

## 3.3 账簿管理闭环

补齐以下标签页真实内容：

- 序时账
- 科目汇总表
- 科目余额表
- 总账
- 明细账
- 收支明细表
- 多栏账
- 财务日志

每个标签页都必须有：

- 查询条件
- 查询按钮
- 列表结果
- 导出按钮
- 打印按钮

## 3.4 期末处理闭环

至少补通以下链路：

1. 执行结转试算
2. 展示试算结果
3. 执行月结
4. 状态切换为已结账
5. 执行反结账
6. 状态切换回未结账

## 4. 资产管理整改

## 4.1 资产列表闭环

至少补通以下链路：

1. 新增资产
2. 查看资产详情
3. 编辑资产
4. 触发资产变更
5. 触发资产处置
6. 生成凭证

## 4.2 资产折旧闭环

至少补通以下链路：

1. 一键计提折旧
2. 显示折旧结果
3. 生成折旧凭证
4. 跳转到凭证详情

## 4.3 资产结账闭环

至少补通以下链路：

1. 点击结账
2. 弹出确认框
3. 结账后状态变化
4. 点击反结账
5. 状态恢复

## 5. 合同管理整改

## 5.1 合同列表闭环

至少补通以下链路：

1. 新增合同
2. 查看合同详情
3. 编辑合同
4. 进入收付款登记
5. 保存收付款记录
6. 发起验收
7. 发起续签或终止

必须补齐：

- 合同详情页
- 收付款登记弹窗
- 验收弹窗
- 续签弹窗
- 终止确认框
- 取消验收入口
- 取消终止入口

## 5.2 合同变更闭环

至少补通：

1. 点击新增变更
2. 填写变更内容
3. 保存后返回变更列表
4. 变更记录可查看详情

## 5.3 合同报表闭环

补齐：

- 收款报表表格明细
- 付款报表表格明细
- 收付款总览表格
- 点击图表后可查看明细

## 6. 在线审批整改

## 6.1 审批中心闭环

至少补通以下两条：

- 待审批 -> 通过 -> 状态变更为已通过
- 待审批 -> 驳回 -> 状态变更为已驳回

必须补齐：

- 审批详情页
- 审批意见输入框
- 审批完成提示

## 6.2 支付管理闭环

至少补通：

1. 新增支付单
2. 提交支付单
3. 在列表中看到待支付
4. 执行支付
5. 状态变更为已支付

## 6.3 历史审批闭环

必须补齐：

- 历史审批详情页
- 审批路径展示
- 与原审批单的关联回查

## 7. 银农直联整改

## 7.1 银行接口设置闭环

至少补通：

1. 新增接口
2. 测试连接
3. 展示连接成功或失败结果

## 7.2 交易查询闭环

至少补通：

1. 选择账户
2. 查询余额
3. 展示余额卡片
4. 查询交易
5. 展示交易流水表

## 7.3 账户和往来闭环

必须补齐：

- 账户管理中的余额查询页
- 开户银行页
- 往来管理中的往来个人页
- 电子回单页

## 8. 本步交付标准

完成本步后，至少能在原型中完整演示以下 5 条链路：

1. 日记账 -> 审核 -> 生成凭证
2. 资产新增 -> 折旧 -> 生成凭证 -> 结账
3. 合同新增 -> 收付款 -> 验收
4. 审批单 -> 通过 / 驳回
5. 银行接口 -> 查询余额 / 查询交易

## 9. AI 执行提示

让 AI 在这一阶段不要分散精力做美化，优先完成最小业务闭环。原则是：

- 每个链路至少有一条演示数据
- 每个链路必须能看到状态变化
- 每个链路必须能从入口走到结果页
