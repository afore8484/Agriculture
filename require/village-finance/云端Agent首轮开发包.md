# 村委财务事务管理系统云端 Agent 首轮开发包

## 1. 开发目标

首轮仅开发 `village-finance` 后端第一阶段最小闭环。

技术前提：
1. Java 17+
2. RuoYi-Vue-Pro
3. PostgreSQL
4. 设计上兼容金仓数据库

目标能力：
1. 账套
2. 会计期间
3. 会计科目
4. 凭证
5. 银行账户
6. 现金账户
7. 银行日记账
8. 现金日记账
9. 内部转账
10. 出纳对账
11. 月结
12. 反结账

## 3. 首轮输出要求

云端 Agent 每轮应至少产出：

1. 已完成功能清单
2. 已实现接口清单
3. 已落地数据对象清单
4. 测试或验证结果
5. 当前阻塞项
6. 下一轮建议

## 4. 开发边界

本轮不包含：
1. 报表中心完整实现
2. 资产管理完整实现
3. 合同管理完整实现
4. 在线审批完整实现
5. 银农直联完整实现
6. 基层党建完整实现

说明：
- 若首轮过程中需要引用这些模块，只允许建立必要关联，不扩展完整实现。

## 5. 数据库兼容要求

1. 表结构和 SQL 设计优先采用 PostgreSQL 与金仓可兼容写法。
2. 避免无必要引入高风险专有特性。
3. 命名、索引、默认值、时间类型选择需考虑未来迁移成本。

## 6. 完成判定

首轮完成必须满足：

1. 账套、期间、科目可被正常查询和使用
2. 凭证具备列表、详情、新增、修改、审核、反审核、删除、回收站能力
3. 银行/现金账户可维护
4. 银行/现金日记账可查询与新增
5. 内部转账可执行并留痕
6. 对账结果可生成并查询
7. 月结与反结账可执行
8. 结果能回到需求梳理文档做覆盖核验
