# 村委财务事务管理系统后台开发任务清单

## 1. 编制依据

本清单仅基于以下三份正式基线文档整理：

1. `docs/village-finance/data/数据结构文档.md`
2. `docs/village-finance/interface/接口文档.md`
3. `require/village-finance/村委财务事务管理系统-需求梳理.md`

执行原则：
- 后端开发以数据结构文档和接口文档为主。
- 开发完成后的功能覆盖校验，以需求梳理文档为准。
- 前端原型工程仅作交互参考，不作为新增需求依据。

## 2. 总体范围

需求梳理文档当前模块统计：
- 财务中心：126 条
- 报表中心：24 条
- 资产管理：20 条
- 合同管理：22 条
- 在线审批：12 条
- 银农直联：7 条
- 基层党建：9 条

接口文档当前定义接口总数：65 个。

## 3. 开发任务分层

### 3.1 基础层任务
1. 建立统一组织、用户、角色、权限、数据字典基础模型。
2. 建立账套、会计期间、会计科目、科目余额、期初余额基础财务模型。
3. 建立统一审计字段规范：创建人、创建时间、更新人、更新时间、删除标识。
4. 建立统一状态枚举规范：启用/停用、草稿/已审核、有效/无效、结账/反结账等。
5. 建立统一校验规则：账号、金额、日期、唯一性、期间状态、权限校验。
6. 建立统一操作日志和审核日志机制。
7. 建立统一导出任务、打印配置、附件存储、文件下载能力。
8. 建立统一分页、筛选、排序、模糊搜索能力。

### 3.2 模块层任务
1. 财务核心模块
2. 资金模块
3. 报表与账簿模块
4. 资产管理模块
5. 合同管理模块
6. 在线审批模块
7. 银农直联模块
8. 基层党建模块
9. 期末处理与结账模块

### 3.3 验收层任务
1. 逐模块对照需求梳理文档检查功能覆盖。
2. 对照接口文档检查接口是否齐全、出入参是否一致。
3. 对照数据结构文档检查表结构、字段和关联是否落地。
4. 对关键流程进行业务闭环验证。

## 4. 模块开发任务清单

### 4.1 基础设置与主数据
对应数据结构：组织机构、用户、角色、权限、数据字典、账套、会计期间、会计科目、科目余额、期初余额。
对应接口：5 个。

任务项：
1. 实现账套管理接口。
2. 实现会计期间查询能力。
3. 实现会计科目树和辅助核算配置查询能力。
4. 实现打印设置查询与保存能力。
5. 实现组织、用户、角色、权限基础数据初始化方案。
6. 实现会计期间状态控制，作为凭证、结账、报表的前置校验。

### 4.2 财务核心与凭证模块
对应需求：财务中心主干能力。
对应数据结构：凭证主表、凭证明细表、附件表、模板表、审核日志、回收站、打印配置、操作日志、辅助核算分录等。
对应接口：12 个。

任务项：
1. 实现凭证列表、详情、新增、修改、删除接口。
2. 实现凭证审核、反审核接口。
3. 实现凭证附件上传接口。
4. 实现凭证导出接口。
5. 实现回收站列表、还原、彻底删除接口。
6. 实现凭证编号规则与自动编号逻辑。
7. 实现凭证明细、借贷平衡、期间状态、审核状态等校验逻辑。
8. 实现凭证审核日志与操作日志记录。
9. 实现凭证与辅助核算、附件、打印配置的关联处理。
10. 补齐凭证模板与模板套用逻辑。

### 4.3 资金模块
对应需求：出纳管理、银行日记账、现金日记账、资金汇总、收支分类、账户管理、收入模板、支出模板、内部转账、出纳对账、初始化模板。
对应数据结构：银行账户、现金账户、银行日记账、现金日记账、内部转账、出纳对账、收入模板、支出模板、初始化模板及其版本/权限/统计等。
对应接口：22 个。

任务项：
1. 实现银行账户、现金账户新增与列表接口。
2. 实现银行日记账、现金日记账查询与新增接口。
3. 实现账户汇总、收支分类汇总接口。
4. 实现收入模板、支出模板查询与保存接口。
5. 实现内部转账查询、新增、删除、导出接口。
6. 实现出纳对账结果与待对账凭证查询接口。
7. 实现初始化模板查询与保存接口。
8. 实现资金模块导出、打印、日志能力。
9. 实现资金业务到凭证的自动生成或关联机制。
10. 落实数据同步、一致性和资金类强校验规则。

### 4.4 报表与账簿模块
对应需求：资产负债表、收益及收益分配表、科目汇总表、做账进度、账簿查询、导出、打印、统计分析。
对应数据结构：账簿日志、多栏账配置、财务收支情况一览表快照、报表定义、报表行次、报表公式、报表快照、统计分析结果、导出记录、打印配置、做账进度快照等。
对应接口：6 个。

任务项：
1. 实现序时账、科目余额表、总账、明细账查询接口。
2. 实现财务报表生成与查询接口。
3. 实现报表导出接口。
4. 实现报表快照和账簿快照生成机制。
5. 实现做账进度统计与快照生成能力。
6. 实现报表公式计算和取数逻辑。
7. 实现打印配置、导出记录、账簿日志留痕。

### 4.5 资产管理模块
对应需求：资产分类、资产卡片、折旧、变更、处置、盘点。
对应数据结构：资产分类、资产卡片、资产折旧、资产变更、资产处置、资产盘点、资产盘点明细。
对应接口：4 个。

任务项：
1. 实现资产卡片列表与新增接口。
2. 实现资产变更记录查询接口。
3. 实现资产盘点列表接口。

### 4.6 合同管理模块

对应数据结构：合同主表、合同变更、合同收付款、合同验收、合同终止、合同续签、合同附件、合同操作日志。
对应接口：3 个。

### 4.7 在线审批模块

## 6. 每阶段交付要求

1. 数据层
完成对应表结构、主外键、索引、状态字段、日志字段。

2. 服务层
完成业务校验、状态流转、事务处理、日志留痕。

3. 接口层
完成接口文档对应接口、统一返回结构、异常码。

4. 验证层
完成接口测试、关键流程测试、需求覆盖校验。

## 7. 验收口径

1. 不允许新增需求。
2. 不允许跳过需求梳理中的既有能力。
3. 不允许接口与数据结构文档脱节。
4. 每完成一个模块，必须从以下三个维度核对：
   - 是否实现了对应表结构
   - 是否实现了对应接口
   - 是否满足了需求梳理中的功能范围

## 8. 当前最小开发闭环

如果现在就开始进入后端开发，最小闭环建议先做：

1. 账套、期间、科目
2. 凭证
3. 银行/现金账户
4. 银行/现金日记账
5. 内部转账
6. 对账
7. 报表基础查询
8. 月结/反结账

这 8 项完成后，财务主线才算真正可运行。
