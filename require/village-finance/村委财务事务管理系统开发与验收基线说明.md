# 村委财务事务管理系统开发与验收基线说明

## 1. 文档角色

### 1.1 后端开发主依据
后端开发以以下两份文档为主依据：

1. 数据结构文档
路径：`docs/village-finance/data/数据结构文档.md`

2. 接口文档
路径：`docs/village-finance/interface/接口文档.md`

说明：
- 数据库设计、表结构、字段设计、实体关系，以数据结构文档为主。
- 接口路径、入参、出参、交互规则，以接口文档为主。
- 若前端原型与上述两份文档不一致，以这两份文档为准。

### 1.2 开发成果检验依据
开发完成后的功能核验、范围校验、需求覆盖校验，以以下文档为准：

1. 需求梳理文档
路径：`require/village-finance/村委财务事务管理系统-需求梳理.md`

说明：
- 该文档用于检验开发成果是否满足需求。
- 不用于替代数据结构文档和接口文档。
- 不允许新增需求，不允许跳过需求项。

### 1.3 原型工程定位
前端原型工程路径：`modules/village-finance/frontend`

说明：
- 原型工程仅作为页面交互和展示参考。
- 原型工程不作为数据库设计主依据。
- 原型工程不作为接口定义主依据。
- 当原型与正式文档冲突时，以正式文档为准。

## 2. 文档优先级
文档优先级固定如下：

1. 数据结构文档
2. 接口文档
3. 需求梳理文档
4. 前端原型工程

解释：
- 第1、2项决定怎么开发。
- 第3项决定开发结果是否满足需求。
- 第4项仅用于补充页面理解，不用于覆盖正式设计结论。

### 3.1 正式开发文档
- `docs/village-finance/data/数据结构文档.md`

### 3.2 正式需求校验文档

### 3.3 来源留档
- `docs/village-finance/source/imported/村委财务事务管理系统.md`

- `docs/village-finance/source/imported/需求梳理与数据结构映射说明.md`

## 4. 执行规则
1. 后续后端开发讨论，默认先看数据结构文档和接口文档。
2. 需要核验交付是否满足业务范围时，默认对照需求梳理文档。
3. 需要理解页面入口、字段展示、交互行为时，可参考前端原型工程。
4. 若发现三者冲突，先记录冲突点，再以正式文档为准，不直接跟随原型修改结论。
