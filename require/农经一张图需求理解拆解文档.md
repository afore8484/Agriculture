# 农经一张图需求理解拆解文档

> 依据需求源：`data/requirements/nongjing-map/nongjing_map_requirements.json`（242条）
> 说明：本文只做“需求理解拆解”，用于后续原型图校验；不新增模块，不输出开发实施方案。

## 一、模块整体理解

### 1. 农经一张图的整体定位
- 这是一个面向“村集体经营管理”的综合观察面板，不是单一业务系统页面。
- 它的核心价值是：在统一的区划与时间口径下，把财务、资产、资源、合同、成员、股权、预警放到同一认知框架中查看。
- 从需求表看，重点不是“录入”，而是“统计展示、对比分析、异常识别、明细钻取”。

### 2. 总览页应承担的作用
- 提供统一入口：清晰进入 7 个二级子模块。
- 提供统一上下文：同一套筛选条件（区划、时间等）驱动各专题页。
- 提供全局态势：在进入专题前，用户能先看到总体规模与风险信号（例如组织规模、月结/账套异常、预警态势）。
- 提供导航连续性：用户在专题页返回总览时，不丢失筛选上下文。

### 3. GIS 在本模块中的作用
- GIS 是“横切能力”，不是第 8 个业务模块。
- GIS 的作用是把“区划维度”可视化并联动到各专题：
  - 地图选区 -> 更新全局筛选；
  - 各模块图表/列表 -> 可反向定位到区划；
  - 同一行政区在不同模块中的数据可对齐查看。
- 原型上，GIS 应表现为各页面可调用的地图联动区或地图抽屉，而非独立主题页。

### 4. 统一查询维度和联动逻辑
- 统一查询维度（建议按需求显式呈现）：
  - 行政区划：省/市/区县/乡镇/村。
  - 时间维度：日期区间、月度、年度。
  - 专题维度：如收入阈值（5万/10万/20万）、合同类型、预警类型、处理进度等。
- 联动逻辑（用于原型校验）：
  - 先选区划和时间，再看汇总指标；
  - 点击汇总指标/图表分段，进入对应明细列表；
  - 明细可继续钻取到对象详情（村、合同、成员、预警记录等）；
  - 返回上一级时筛选条件保持不变。

## 二、总览页理解

### 1. 页面目标
- 在一个页面内完成三件事：
  - 告诉用户“现在看的是哪一片区、哪个时间段”；
  - 告诉用户“7个专题入口在哪里”；
  - 告诉用户“是否存在需要优先关注的异常信号”。

### 2. 页面组成区域
- 全局筛选区（区划、时间、查询按钮）
- 总体概览区（规模类指标）
- 模块入口区（7个二级子模块卡片）
- GIS联动区（地图选区）
- 风险提示区（预警和处理进度摘要）

### 3. 每个区域展示什么内容
- 全局筛选区：当前区划、开始/结束时间、统计口径。
- 总体概览区：与财务首页一致的“组织规模/基础异常”类信息（如区县数、乡镇数、村集体数、未月结数、未建账套数），并显示当前时间信息。
- 模块入口区：
  - 必须完整显示 7 个模块名称：财务一张图、资产一张图、资源一张图、合同一张图、成员一张图、股权一张图、预警监督。
  - 每个入口可直接进入专题页。
- GIS联动区：地图点击后应刷新页面上下文，不是静态图片。
- 风险提示区：至少展示预警总量/处理状态概览，作为进入“预警监督”的前置判断。

### 4. 用户从总览页如何进入各专题页
- 方式1：点击模块卡片进入。
- 方式2：点击概览区或风险区的指标快捷进入对应专题（例如“未月结”进入财务相关明细、“预警处理中”进入预警处理中列表）。
- 两种方式都应继承当前筛选条件。

### 5. 总览页待确认问题
- “总览页指标”与“财务首页指标”是同一套还是只保留摘要版？
- 天气信息是否必须长期保留在总览页，还是仅财务首页需要？
- 总览页是否允许直接展示跨模块排行（如高风险区划 TopN）？

## 三、7个二级子模块逐个拆解

## 3.1 财务一张图
### 模块定位
- 以“村集体经营收支 + 资产负债 + 收益分配”为核心的财务总览与穿透分析模块。

### 核心业务对象
- 村集体组织、收入阈值分组村（高于/低于5万10万20万）、资产负债表项目、收益分配科目。

### 主要展示内容
- 首页基础信息：日期时间、天气、区划筛选、组织规模、未月结、未建账套。
- 收入分档分析：达到/高于/低于5万10万20万的数量与占比。
- 分档明细穿透：按区划查看高低分组村明细；支持查看、查询、导出。
- 收入构成项：总收入、经营收入、补助收入、投资收益、其他收入。
- 财务指标：货币资金、应收款、存货、固定资产、应付款、经营收支、管理费用、本年收益、未分配收益。
- 资产负债分析：流动/非流动资产、流动/非流动负债、所有者权益、年初/年末对比、合计变动。
- 收益及分配：本年收益、税费、净收益、可分配收益、公积公益金、分配给农户、年末未分配收益。
- 多项变动分析：资产、负债、收入、支出、权益等变动趋势。

### 典型页面组成
- 页面A：财务首页总览（指标卡 + GIS联动 + 快捷入口）
- 页面B：收入分档页（5/10/20万阈值切换）
- 页面C：分档明细页（高于/低于切换、列表、导出）
- 页面D：财务指标页（指标图表+口径说明）
- 页面E：资产负债分析页（报表+年初年末对比）
- 页面F：收益分配页（收益分配表+变动分析）

### 可能的筛选条件
- 区划、时间区间、月/年、收入阈值（5/10/20万）、高于/低于、收入类型、报表科目。

### 从汇总到明细的钻取路径
- 财务首页指标 -> 收入分档 -> 高/低分组列表 -> 区划详情 -> 村级明细。
- 财务指标卡 -> 资产负债变动 -> 报表详情 -> 科目级明细。
- 收益分配总览 -> 科目金额 -> 分配对象明细（村/农户层面）。

### 与其他模块的关联关系
- 与合同：应收/应付与合同收付款联动。
- 与资产：固定资产、生物资产与资产模块口径对齐。
- 与股权：收益分配与分红口径联动。
- 与预警：月结异常、负债风险触发预警。

### 当前待确认问题
- “达到5/10/20万”与“高于/低于5/10/20万”是否需要同时保留并同屏展示？
- 导出范围是当前筛选结果，还是固定字段全量导出？
- 收益分配是否需要下钻到农户级分配明细。

## 3.2 资产一张图
### 模块定位
- 展示村集体资产规模、结构和区域排名，强调资产价值对比。

### 核心业务对象
- 资产总值、经营性资产、非经营性资产、资产类型、区划内排名村。

### 主要展示内容
- 区划资产价值总额统计。
- 经营性资产原值统计与图表。
- 非经营性资产原值统计与图表。
- 资产类型占比（固定资产/流动资产/无形资产等）。
- 村集体总收入 Top10 且 >=100万（需求中明确出现，需保留为页面展示项）。
- 各区域资产价值 Top5 村。

### 典型页面组成
- 资产总览页（资产规模卡片）
- 资产结构页（经营性/非经营性 + 类型占比图）
- 区域排行页（Top10、Top5、榜单联动）

### 可能的筛选条件
- 区划、时间、资产类型、经营性/非经营性、排行维度。

### 从汇总到明细的钻取路径
- 区划资产总值 -> 资产类型占比 -> 区域排行 -> 村级资产明细。

### 与其他模块的关联关系
- 与财务：固定资产、无形资产等口径需一致。
- 与资源：土地资源可能形成资产价值基础。
- 与合同：资产租赁合同影响资产收益与状态。
- 与预警：资产结构异常可触发预警。

### 当前待确认问题
- “总收入Top10且>=100万”放在资产模块是否为业务方刻意设计，是否保留原样。
- 资产价值是期末值、平均值还是区间累计值。

### 从汇总到明细的钻取路径

- 人员效能：受理员送办效能、村会计处理效能。

### 典型页面组成

- 风险分布页
- 区域排名页
- 处理进度页

### 与其他模块的关联关系

## 四、模块之间的关系理解

- 成员 -> 股权：成员身份、成员结构变化会直接影响成员股、村龄股及分红对象。
- 股权 -> 财务：股权分红来源于收益分配，需与财务“可分配收益/未分配收益”一致。
- 合同 -> 财务：合同产生的应收应付进入财务科目，影响现金流与负债。
- 资产 -> 财务：资产原值、折旧、净值是资产负债表的重要来源。
- 资源 -> 资产/合同：资源（尤其土地）既是资产基础，也常通过合同形成经营收益。
- 预警 -> 全模块：预警不是孤立页面，而是对各模块异常信号的统一监督闭环。
- GIS -> 全模块：所有模块共享同一空间定位能力，使跨模块对比具备同一地理参照。

## 五、原型校验点（用于判断“画得对不对”）

### 1. 结构校验
- 是否严格只有 7 个二级子模块入口，名称完全一致。
- 是否把 GIS 作为横切能力呈现，而不是独立业务主题页。
- 是否有统一筛选区，并在模块切换后保持筛选上下文。

### 2. 内容校验
- 财务页是否覆盖“收入分档 + 高低明细 + 财务指标 + 资产负债 + 收益分配 + 变动分析”。
- 资产页是否同时出现“经营性/非经营性/类型占比/Top排行”。
- 资源页是否至少覆盖“农用地/建设用地/未利用地/总面积/资源占比”。
- 合同页是否同时出现“数量+金额+本期应收应付+累计应收应付”。
- 成员页是否覆盖“性别、年龄、民族、成员转化”四类结构分析。
- 股权页是否覆盖“类型占比+类型金额+Top10+年度分红对比”。
- 预警页是否覆盖“规则配置+风险占比+区域排名+处理进度+效能评估”。

### 3. 交互校验
- 每个汇总图表是否可点击进入对应明细列表（不是静态图）。
- 明细列表是否支持按当前筛选继续过滤。
- 返回上级页面后，筛选条件和定位状态是否保持。

### 4. 钻取链路校验
- 是否都存在“汇总 -> 分组 -> 列表 -> 详情”的完整路径。
- 财务分档是否具备“高于/低于5万10万20万”的双向钻取。
- 预警是否具备“类型 -> 区域 -> 状态 -> 责任人”的闭环追踪路径。

### 5. 跨模块一致性校验
- 同一行政区在不同模块的数据入口是否一致（区划口径一致）。
- 时间维度是否统一（例如月度、年度在各模块表达一致）。
- 指标命名是否避免同名异义（如“总收入”“总分红”“总股值”边界清晰）。

### 6. 待确认问题汇总（用于原型前置澄清）
- 财务模块阈值规则中“达到”“高于”“低于”的判定边界是否统一（是否含等于）。
- 资产模块中的“总收入Top10且>=100万”是否保留在资产页。
- 成员模块民族展示是否固定民族项还是动态配置。
- 股权模块“总股占比”的业务含义是否需要单独解释。
- 预警模块处理率/已办率/送办率分母口径是否统一。
