# 三农金融平台二期需求调研问题预期答案

## 使用说明

本文件用于约束调研输出质量。前提是：二期需求范围已确定，所有既定需求都需要落地。本文件关注的不是功能取舍，而是要求业务方把规则、流程、口径、边界、数据和验收标准讲清楚。

## 预期答案

### 1. 当前已确定需求中，二期建设要落地的完整业务流程是什么
- 预期回答应形成“流程步骤 + 处理角色 + 输入输出 + 结束条件”。
- 示例结构：
  - 发起：由村级或主体端录入业务数据。
  - 审核：由乡镇或管理部门审核。
  - 复核/监管：由县级主管部门监督或复核。
  - 输出：形成台账、报表、预警或审批结果。

### 2. 每类使用角色在各模块中的具体职责和操作边界是什么
- 预期回答应形成“角色名称 + 所属层级 + 可见范围 + 可操作动作”。
- 示例结构：
  - 村委财务人员：录入、修改、提交财务数据。
  - 乡镇审核人员：审核、退回、汇总查看。
  - 县级监管人员：查询、统计、监督、导出。
  - 金融机构人员：查看主体画像、筛选客户、跟踪授信。
  - 审计人员：调取资料、登记疑点、形成报告。

### 3. 每个模块的核心业务页面分别承载什么任务
- 预期回答应形成“页面名称 + 页面目标 + 关键操作 + 输出结果”。
- 示例结构：
  - 凭证管理：完成凭证录入、修改、审核、反审核、附件管理。
  - 合同管理：完成合同登记、变更、履约跟踪、到期预警。
  - 资产管理：完成资产台账维护、状态变更、处置审批、统计分析。
  - 主体监管页面：完成主体查询、经营分析、风险预警、金融服务支撑。

### 4. 各模块的关键业务对象及其关联关系是什么
- 预期回答应形成“对象名称 + 主标识 + 关联对象 + 业务含义”。
- 示例结构：
  - 村集体组织关联资产、合同、财务凭证、收入台账。
  - 农户关联家庭成员、补贴信息、档案信息。
  - 经营主体关联客户、商品、项目、进销存、信贷申请。
  - 审计任务关联审计对象、资料、疑点、报告。

### 5. 关键字段、台账和单据的业务口径是什么
- 预期回答应形成“字段/单据 + 口径定义 + 填写规则 + 是否可修改”。
- 示例结构：
  - 凭证编号：按年度或组织维度自动生成。
  - 合同金额：含税/不含税口径需明确。
  - 资产状态：在用、出租、闲置、处置等状态需标准化。
  - 收入口径：按自然年还是会计年度统计需明确。

### 6. 数据录入、审核、复核、反审核、删除、恢复等动作的规则是什么
- 预期回答应形成“动作名称 + 触发角色 + 前置条件 + 后置状态 + 日志要求”。
- 示例结构：
  - 审核：只能对已提交数据执行，审核后锁定关键字段。
  - 反审核：仅特定角色可发起，需记录原因。
  - 删除：逻辑删除进入回收站，彻底删除需更高权限。
  - 恢复：仅在未归档状态下可恢复。

### 7. 系统需要采集或接入哪些外部数据，这些数据分别服务于什么业务动作
- 预期回答应形成“数据名称 + 来源系统 + 使用动作 + 更新频率 + 责任单位”。
- 示例结构：
  - 农户档案：用于主体识别、家庭关联、信息补全。
  - 补贴信息：用于画像和核验。
  - 三资余额：用于财务对账、统计分析。
  - 经营主体基础数据：用于监管查询、金融辅助。
- 预期不应再回答“是否要接入”，而应回答“接入后具体怎么用”。

### 8. 统计分析、报表和预警规则应如何定义
- 预期回答应形成“指标名称 + 计算口径 + 展示维度 + 更新频率 + 责任角色”。
- 示例结构：
  - 财务报表：按村、乡镇、区县分级汇总。
  - 经营分析：按主体类型、行业、时间维度统计。
  - 预警规则：合同到期、长期未入账、异常负债、经营异常等。
  - 闭环要求：预警产生后可指派、处理、反馈、归档。

### 9. 异常场景和特殊业务场景怎么处理
- 预期回答应形成“异常类型 + 触发条件 + 处理动作 + 是否留痕”。
- 示例结构：
  - 历史数据补录：允许补录但需标记来源和时间。
  - 重复数据：系统给出校验提示并阻断或允许覆盖。
  - 审批中断：支持撤回、转派或重新发起。
  - 附件缺失：允许暂存还是禁止提交需明确。

### 10. 二期功能上线后的验收标准和成功标准是什么
- 预期回答应形成“验收场景 + 验收指标 + 通过标准”。
- 示例结构：
  - 场景：凭证闭环、合同闭环、资产闭环、收入填报闭环、经营监管闭环、审计闭环。
  - 指标：功能覆盖、数据准确率、报表一致性、日志完整性、接口成功率、预警有效性。
  - 结果：形成可执行的验收用例和上线清单。

## 调研结果回填要求
- 每个问题至少沉淀一条明确结论、一条业务规则、一条待确认事项。
- 对涉及字段、报表、预警、审批链的问题，必须补充到可以用于原型、数据结构和测试用例设计的颗粒度。
- 不再记录“是否建设”，只记录“如何实现”和“如何验收”。
